Ubuntu one-command install

Lowest-friction Linux path. Fetches the signed dinero-core .deb (below), verifies its SHA256 against the digest GitHub publishes for the asset, installs via apt, enables dinero.service, opens P2P port 20999/tcp on ufw when active, then waits for first peer connection. Ubuntu 24.04+ amd64, ~90 seconds end-to-end. Read it before piping to sudo — source linked below.

curl -fsSL https://dinerolabs.org/install.sh | sudo bash

Run a node

Datadirs follow the native convention for each install mode. Linux manual-user mode intentionally keeps the standard dot-directory path.

Platform / mode Default datadir Use case
Windows desktop / Core zip %APPDATA%\Dinero Windows application-data convention
macOS desktop ~/Library/Application Support/Dinero macOS application-data convention
Windows Server installer C:\ProgramData\Dinero Windows Service convention; survives uninstall
Linux dinero-core .deb /var/lib/dinero FHS/systemd daemon under the dinero service user
Linux operator tarball / developer ~/.dinero Standard Linux CLI/daemon dot-directory convention
Linux GUI (AppImage / Qt tarball / dinero-qt-desktop .deb) ~/.dinero GUI auto-starts the bundled daemon in the user's home

Linux dinero-core .deb quick start

sudo dpkg -i dinero-core_8.0.0.rc6-1_amd64.deb
sudo systemctl enable --now dinero
sudo -u dinero dinero-cli -datadir=/var/lib/dinero getblockchaininfo

Linux operator tarball quick start

tar -xzf dinero-v8.0.0-rc6-linux-x86_64.tar.gz
cd dinero-v8.0.0-rc6-linux-x86_64
sha256sum -c SHA256SUMS.txt
./bin/dinerod                          # mainnet, ~/.dinero
./bin/dinero-cli getblockchaininfo

Linux AppImage quick start

chmod +x dinero-v8.0.0-rc6-linux-x86_64.AppImage
./dinero-v8.0.0-rc6-linux-x86_64.AppImage    # GUI auto-starts the bundled daemon

# If libfuse2 is missing: sudo apt install libfuse2
# Or extract + run without FUSE:
./dinero-v8.0.0-rc6-linux-x86_64.AppImage --appimage-extract
squashfs-root/AppRun

What is not stable yet

  • Windows v8 installers (user + server) are not yet Authenticode-signed — SmartScreen may warn. Verify the SHA256 above and click "More info" → "Run anyway".
  • Windows + Linux are at v8.0.0-rc6 with real UPnP / NAT-PMP runtime support; macOS is still at v8.0.0-rc5 (rc6 mac rebuild pending). The rc5 mac assets remain Apple-notarized and safe to run; they just lack the rc6-side port-mapping linkage.
  • dinero-gpu-miner ships on Windows but its OpenCL runtime path is not yet validated on MSVC; treat as experimental. Not shipped on Linux for rc5 (CUDA toolkit unavailable in the build environment).
  • Stable promotion waits for the next final-release ceremony.
